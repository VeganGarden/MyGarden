## 1. 气候餐厅核心设计

### 1.1 气候餐厅定义

**气候餐厅（Climate Restaurant）**是指通过认证的低碳餐饮机构，致力于通过提供低碳菜品、优化供应链、减少食物浪费等方式，实现可量化的碳减排目标，并承担社会责任，推动餐饮行业向可持续发展转型。

### 1.2 核心要素

#### 1.2.1 认证体系

为降低行业门槛、加速概念普及，本方案采用“**单级认证 + 成长激励**”模式：通过统一认证标准，让餐厅先迈入体系，再通过数据驱动的成长任务持续提升。

**达标项（必须满足）**：

| 维度           | 达标阈值与要求                                                            | 核查方式                  |
| -------------- | ------------------------------------------------------------------------- | ------------------------- |
| 低碳菜品占比   | 低碳及以下菜品占比 ≥40%，且核心菜品须提供碳足迹标签                       | 平台碳核算系统 + 菜单抽查 |
| 食材与供应链   | 本地（≤100km）或可追溯低碳食材占比 ≥30%，关键食材需提供来源证明           | 溯源模块 + 供应商资料     |
| 能源与运营     | 建立能源使用台账，年度能源强度下降 ≥10% 或提供绿色能源使用证明            | 能源账单/设备数据         |
| 食物浪费管理   | 建立浪费监测流程，月度浪费减量目标 ≥15%，并提供数据记录                   | 浪费监测报表 + 现场核查   |
| 社会传播与教育 | 提供不少于 3 项低碳倡导举措（如顾客教育、员工培训、公益活动），并形成记录 | 活动记录 + 顾客反馈       |

**成长激励项（可选）**：

- 进阶低碳行动：如引入零碳能源、供应链碳数据共享、碳中和项目等，获得数字勋章展示
- 创新联动项目：与政府、公益、品牌合作的气候行动案例，可解锁专题报道资源
- 数据透明度：实现高频数据上报、通过第三方审计的餐厅，在平台排名与曝光上获得优先

**认证评估机制**：

1. **资料提交**：餐厅上传菜单、供应链、能源与浪费管理数据
2. **系统评估**：平台基于碳核算与数据校验自动评分，生成达标报告
3. **人工抽检**：对高风险项及关键达标项进行抽检或远程/现场核查
4. **认证发布**：达标餐厅获得统一认证证书与数字标识，进入年度复审周期
5. **持续运营**：通过数据看板跟踪达标项完成度，完成成长激励任务可解锁勋章、曝光位

**认证标准**（五大维度）：

1. **低碳菜品占比**（核心指标）

   - 定义：碳足迹低于同类传统菜品平均值的菜品占比
   - 计算方法：基于食材数据库计算每道菜的碳足迹
   - 评分权重：40%

2. **本地食材占比**

   - 定义：来自本地（100km 内）的食材占比
   - 价值：减少运输碳排放
   - 评分权重：20%

3. **有机食材占比**

   - 定义：获得有机认证的食材占比
   - 价值：减少化肥农药使用，保护土壤
   - 评分权重：15%

4. **食物浪费减少**

   - 定义：相比行业平均水平的食物浪费减少比例
   - 价值：减少全生命周期碳排放
   - 评分权重：15%

5. **能源效率**
   - 定义：餐厅运营的能源使用效率
   - 价值：减少运营碳排放
   - 评分权重：10%

#### 1.2.2 碳足迹核算

**核算范围**（全生命周期）：

1. **食材生产阶段**

   - 种植/养殖过程碳排放
   - 化肥、农药、饲料等投入品碳排放
   - 土地利用变化碳排放

2. **运输阶段**

   - 食材从产地到餐厅的运输碳排放
   - 距离、运输方式（公路/铁路/航空）影响

3. **加工阶段**

   - 食材加工、烹饪过程碳排放
   - 能源消耗（燃气、电力）
   - 包装材料碳排放

4. **消费阶段**

   - 餐厅运营能源消耗
   - 餐具、清洁用品碳排放

5. **废弃阶段**
   - 食物浪费处理碳排放
   - 包装废弃物处理碳排放

**核算方法**：

- **菜品级核算**：每道菜品的完整碳足迹
- **订单级核算**：每笔订单的碳足迹汇总
- **餐厅级核算**：餐厅整体碳足迹统计
- **对比基准**：与同类传统餐厅对比，计算碳减排量

#### 1.2.3 年度承诺

**碳减排目标**：

- 餐厅需设定年度碳减排目标（kg CO₂e）
- 目标需基于历史数据和业务增长预测
- 目标需可量化、可验证

**参与人次目标**：

- 设定年度服务人次目标
- 鼓励更多消费者参与低碳消费

**承诺履行**：

- 月度跟踪：每月更新碳减排进度
- 季度评估：每季度评估目标完成情况
- 年度审核：年度审核承诺履行情况

#### 1.2.4 品牌标识

**认证标识**：

- 统一的“Climate Restaurant Certified”徽标（实体铭牌 + 数字证书）
- 可在餐厅门口、菜单、宣传材料、线上渠道使用
- 扫码即可查看认证报告、数据透明度、成长勋章等信息

**碳标签**：

- 菜品碳标签：显示每道菜的碳足迹
- 标签样式：红绿灯（高/中/低碳）、碳评分（0-100）、碳数值（kg CO₂e）
- 帮助消费者做出低碳选择

**影响力故事**：

- 餐厅的低碳实践故事
- 碳减排成果展示
- 顾客证言和评价

### 1.3 价值主张

#### 对餐厅

- 🏆 **品牌提升**：获得权威认证，提升品牌形象和竞争力
- 🎯 **精准获客**：触达素食和低碳消费人群
- 💰 **财务激励**：参与碳普惠，获得政府补贴和订单增长
- 📊 **数据驱动**：完整的碳足迹数据，支持经营决策

#### 对消费者

- 🌱 **健康选择**：了解菜品碳足迹，做出健康环保选择
- 💰 **经济激励**：低碳消费获得积分奖励
- 🌍 **社会价值**：参与低碳行动，贡献环保力量
- 📱 **便捷体验**：扫码点餐，自动记录碳减排

#### 对社会

- 🌍 **可量化成果**：可量化的城市级碳减排数据
- 📋 **ESG 案例**：支持 ESG 披露和可持续发展报告
- 👥 **社会动员**：推动公众参与低碳行动
- 🏆 **行业标杆**：推动餐饮行业低碳转型

## 2. 气候餐厅运营模式

### 2.1 运营流程

```
餐厅申请认证
    ↓
提交认证资料（菜单、食材、供应链信息）
    ↓
平台审核（资料审核 + 现场核查）
    ↓
获得“Climate Restaurant Certified”认证
    ↓
上线运营（菜单管理、订单管理、数据统计）
    ↓
月度跟踪（碳减排数据、目标完成情况）
    ↓
年度审核（承诺履行情况、续费认证）
```

### 2.2 核心业务场景

#### 场景 1：认证申请

1. **餐厅注册**：餐厅在平台注册账号
2. **资料提交**：
   - 餐厅基本信息（名称、地址、联系方式）
   - 菜单信息（菜品名称、食材清单、份量）
   - 供应链信息（供应商、食材来源、认证信息）
   - 运营数据（能源消耗、食物浪费情况）
3. **初步评估**：平台自动计算初步认证等级
4. **现场核查**：平台安排现场核查（可选）
5. **认证颁发**：通过审核后颁发认证证书

#### 场景 2：日常运营

1. **菜单管理**：

   - 菜品信息维护（名称、价格、图片、描述）
   - 碳足迹计算（自动计算每道菜的碳足迹）
   - 碳标签设置（选择标签样式和展示方式）
   - 低碳推荐（设置每日特供、时令推荐）

2. **订单管理**：

   - 订单接收（扫码点餐、在线点餐）
   - 订单处理（确认、制作、配送）
   - 订单完成（自动计算碳足迹、同步到用户账户）

3. **数据统计**：
   - 实时数据（今日订单、今日碳减排）
   - 月度数据（月度订单、月度碳减排、目标完成情况）
   - 年度数据（年度订单、年度碳减排、年度报告）

#### 场景 3：用户互动

1. **用户点餐**：

   - 浏览菜单（查看菜品信息、碳标签）
   - 选择菜品（根据碳足迹选择）
   - 下单支付（微信支付、积分支付）

2. **碳减排同步**：

   - 订单完成后自动计算碳足迹
   - 同步到用户"我的花园"账户
   - 用户获得积分奖励

3. **用户评价**：
   - 用户对餐厅和菜品进行评价
   - 评价包含碳减排满意度
   - 评价影响餐厅评分和排名

#### 场景 4：数据报告

1. **月度报告**：

   - 月度碳减排数据
   - 订单统计（订单量、客单价、复购率）
   - 用户分析（用户画像、消费行为）
   - 目标完成情况

2. **年度报告**：
   - 年度碳减排数据
   - ESG 报告（环境、社会、治理）
   - 认证材料（用于续费或升级）
   - 影响力评估

### 2.3 激励机制

#### 对餐厅

1. **认证激励**：

   - 获得认证标识，提升品牌形象
   - 在平台获得推荐展示
   - 获得精准用户触达

2. **财务激励**：

   - 政府补贴（参与碳普惠项目）
   - 订单增长（低碳消费人群）
   - 碳积分收益（可兑换或交易）

3. **数据激励**：
   - 完整的数据报告
   - 经营决策支持
   - ESG 披露支持

#### 对消费者

1. **积分激励**：

   - 低碳消费获得积分
   - 积分可用于商城购物
   - 积分可用于餐厅消费

2. **成就激励**：

   - 解锁成就徽章
   - 参与排行榜
   - 获得社会认可

3. **优惠激励**：
   - 会员折扣
   - 专属优惠券
   - 限时活动

## 3. 数字化软件功能策划

> **数据使用说明**：本节列举的碳排系数、基准值等数值为方案阶段示例，仅用于说明算法和产品逻辑。实际落地需按照“3.1.2.9 碳数据来源与更新机制”与“3.1.2.10 对外数据协同与接口方案”接入权威数据库、第三方核算机构或政府平台数据，并由专业人员完成校准与版本管理。文中计算逻辑参考 ISO 14064、PAS 2050 等通用碳核算方法论，落实时须结合最新标准与监管要求再次验证。

##### 3.1.2.1 基础计算方法（简单模式）

适用于快速计算场景，计算公式如下：

**单食材碳足迹计算**：

```
食材碳足迹 (kg CO₂e) = 碳足迹系数 (kg CO₂e/kg) × 食材重量 (kg)
```

**示例**：计算 200g 菠菜的碳足迹

- 菠菜属于叶菜类，碳足迹系数：0.4 kg CO₂e/kg
- 食材重量：200g = 0.2 kg
- 碳足迹 = 0.4 × 0.2 = 0.08 kg CO₂e

**菜品总碳足迹计算**：

```
菜品碳足迹 = Σ(各食材碳足迹) × 烹饪方式系数
```

**示例**：计算"清炒时蔬"的碳足迹

- 食材 1（菠菜 200g）：0.08 kg CO₂e
- 食材 2（胡萝卜 150g，系数 0.3）：0.3 × 0.15 = 0.045 kg CO₂e
- 食材基础碳：0.08 + 0.045 = 0.125 kg CO₂e
- 烹饪方式（炒）：系数 1.15
- 菜品总碳足迹 = 0.125 × 1.15 = 0.144 kg CO₂e

**碳减排量计算**：

```
碳减排量 = 基准餐碳足迹 - 实际菜品碳足迹
```

**基准值设定**：

- 素食简餐基准：1.0 kg CO₂e
- 素食正餐基准：1.5 kg CO₂e
- 肉食简餐基准：5.0 kg CO₂e
- 肉食正餐基准：7.5 kg CO₂e

**示例**：对比肉食简餐

- 基准碳足迹（肉食简餐）：5.0 kg CO₂e
- 实际菜品碳足迹：0.144 kg CO₂e
- 碳减排量 = 5.0 - 0.144 = 4.856 kg CO₂e

##### 3.1.2.2 高级计算方法（多因子模型）

适用于精确核算场景，采用全生命周期多因子模型，全面考虑食材从生产到消费各环节的碳排放：

**总碳足迹计算公式**：

```
总碳足迹 = 食材基础碳 + 烹饪能耗碳 + 运输碳 + 保存碳 + 季节调整碳
```

**计算流程**：

1. 计算每个食材的基础碳足迹
2. 根据各因子分别计算额外碳排放
3. 累加得到总碳足迹
4. 生成详细分解报告

**各因子计算方法**：

1. **食材基础碳足迹**：

   ```
   食材基础碳 = Σ(食材碳足迹系数 × 食材重量)
   ```

   - 从食材数据库获取每个食材的碳足迹系数（kg CO₂e/kg）
   - 支持查询`ingredients`集合获取精确系数

2. **烹饪能耗碳排**：

   ```
   烹饪碳排 = 食材基础碳 × (烹饪系数 - 1.0)
   ```

   - 烹饪方式系数表：
     | 烹饪方式 | 系数 | 说明 |
     |---------|------|------|
     | 生食/凉拌 | 1.0 | 无需加热 |
     | 蒸 | 1.05 | 低能耗 |
     | 煮/炖 | 1.08-1.12 | 中等能耗 |
     | 炒/煎 | 1.15-1.18 | 中等能耗 |
     | 烤/烘焙 | 1.25-1.28 | 高能耗 |
     | 炸 | 1.35 | 最高能耗 |

3. **运输碳排**：

   ```
   运输碳排 = 食材基础碳 × (运输系数 - 1.0)
   ```

   - 运输距离系数表：
     | 运输类型 | 系数 | 说明 |
     |---------|------|------|
     | 本地食材 | 1.0 | <100km |
     | 国内运输 | 1.15 | 跨省运输 |
     | 国际运输 | 1.35 | 进口食材 |

4. **保存方式碳排**：

   ```
   保存碳排 = 食材基础碳 × (保存系数 - 1.0)
   ```

   - 保存方式系数表：
     | 保存方式 | 系数 | 说明 |
     |---------|------|------|
     | 新鲜 | 1.0 | 新鲜食材 |
     | 冷藏 | 1.05 | 需要冷藏 |
     | 冷冻 | 1.10 | 需要冷冻 |
     | 干货 | 1.08 | 脱水处理 |
     | 罐装 | 1.15 | 加工包装 |

5. **季节调整碳排**：
   ```
   季节碳排 = 食材基础碳 × (季节系数 - 1.0)
   ```
   - 应季食材：系数 = 1.0（无额外碳排）
   - 反季食材：系数 = 1.2（增加 20%碳排）
   - 根据当前季节判断食材是否应季

**完整计算示例**：使用高级方法计算"清炒时蔬"（冬季制作）

**输入数据**：

- 菠菜 200g（本地，新鲜，应季）
- 胡萝卜 150g（跨省运输，新鲜，应季）
- 烹饪方式：炒
- 当前季节：冬季

**计算过程**：

1. **食材基础碳足迹**：

   - 菠菜：0.4 × 0.2 = 0.08 kg CO₂e
   - 胡萝卜：0.3 × 0.15 = 0.045 kg CO₂e
   - 基础碳合计：0.125 kg CO₂e

2. **烹饪能耗碳排**：

   - 烹饪系数（炒）：1.15
   - 烹饪碳排 = 0.125 × (1.15 - 1.0) = 0.01875 kg CO₂e

3. **运输碳排**：

   - 菠菜（本地）：0.08 × (1.0 - 1.0) = 0 kg CO₂e
   - 胡萝卜（跨省）：0.045 × (1.15 - 1.0) = 0.00675 kg CO₂e
   - 运输碳合计：0.00675 kg CO₂e

4. **保存方式碳排**：

   - 菠菜（新鲜）：0.08 × (1.0 - 1.0) = 0 kg CO₂e
   - 胡萝卜（新鲜）：0.045 × (1.0 - 1.0) = 0 kg CO₂e
   - 保存碳合计：0 kg CO₂e

5. **季节调整碳排**：
   - 菠菜（应季）：0.08 × (1.0 - 1.0) = 0 kg CO₂e
   - 胡萝卜（应季）：0.045 × (1.0 - 1.0) = 0 kg CO₂e
   - 季节碳合计：0 kg CO₂e

**总碳足迹**：

- 总碳足迹 = 0.125 + 0.01875 + 0.00675 + 0 + 0 = 0.1505 kg CO₂e
- 四舍五入：0.15 kg CO₂e

**碳排分解占比**：

- 食材基础：83.1%
- 烹饪能耗：12.5%
- 运输：4.5%
- 保存：0%
- 季节调整：0%

**优化建议**：由于运输碳排占比 4.5% < 15%，无显著优化空间。建议保持现有配置。

##### 3.1.2.4 订单级碳足迹计算

**单个订单碳足迹**：

```
订单碳足迹 = Σ(订单中所有菜品的碳足迹)
```

**示例**：计算一个订单的碳足迹

- 菜品 1（清炒时蔬）：0.144 kg CO₂e
- 菜品 2（麻婆豆腐，300g）：1.2 × 0.3 × 1.15 = 0.414 kg CO₂e
- 菜品 3（米饭，200g）：1.4 × 0.2 × 1.08 = 0.302 kg CO₂e
- 订单总碳足迹 = 0.144 + 0.414 + 0.302 = 0.860 kg CO₂e

**订单级碳减排量**：

```
订单减排量 = 订单基准碳足迹 - 订单实际碳足迹
订单基准碳足迹 = 订单中菜品数量 × 同类肉食餐平均碳足迹
```

**示例**：计算订单减排量

- 订单包含 3 道菜
- 基准碳足迹（按肉食简餐计算）：3 × 5.0 = 15.0 kg CO₂e
- 实际订单碳足迹：0.860 kg CO₂e
- 订单减排量 = 15.0 - 0.860 = 14.14 kg CO₂e

##### 3.1.2.5 餐厅级碳足迹统计

**累计碳减排**：

```
累计减排 = Σ(所有订单的碳减排量)
```

**月度/年度碳减排**：

```
期间减排 = Σ(该期间内所有订单的碳减排量)
```

**目标完成率**：

```
完成率 = (期间实际减排量 / 期间目标减排量) × 100%
```

##### 3.1.2.6 碳标签分级标准

根据菜品碳足迹（kg CO₂e）自动分级，帮助消费者快速识别低碳菜品：

| 等级   | 碳足迹范围   | 标签颜色 | 碳评分    | 说明                 |
| ------ | ------------ | -------- | --------- | -------------------- |
| 超低碳 | < 0.5 kg     | 🟢 绿色  | 90-100 分 | 推荐选择，环保优先   |
| 低碳   | 0.5 - 1.0 kg | 🟡 黄色  | 70-89 分  | 良好选择，可优先考虑 |
| 中碳   | 1.0 - 2.0 kg | 🟠 橙色  | 50-69 分  | 一般水平，可适度选择 |
| 高碳   | > 2.0 kg     | 🔴 红色  | 0-49 分   | 碳排较高，建议减少   |

**碳评分计算公式**：

```
碳评分 = max(0, 100 - (菜品碳足迹 / 基准碳足迹) × 50)
```

**示例**：计算"清炒时蔬"的碳评分

- 菜品碳足迹：0.144 kg CO₂e
- 基准碳足迹（素食简餐）：1.0 kg CO₂e
- 碳评分 = max(0, 100 - (0.144 / 1.0) × 50) = max(0, 100 - 7.2) = 92.8 分
- 评级：超低碳（🟢 绿色标签）

**碳足迹标签生成流程**：

1. **数据采集**：餐厅管理员在后台从平台标准食材/配方库选配菜品，补充门店份量、烹饪方式、产地等参数，必要时发起新增申请或接入扫码/AI 识别/供应链数据。
2. **系统核算**：平台碳足迹引擎自动计算每道菜的碳排量与等级，并校验数据完整性、系数有效性。
3. **标签生成**：根据核算结果，系统统一生成标准化的碳足迹标签（红绿灯/数值/评分三种样式），并输出到后台、点餐小程序与认证报告中。
4. **发布与更新**：餐厅在菜单管理中勾选展示方式后，标签实时同步到线下菜单、线上点餐、宣传物料；当食材或核算数据更新时，标签自动刷新并保留历史版本以供追溯。

##### 3.1.2.7 计算优化建议

系统根据计算结果自动生成个性化优化建议，帮助餐厅持续降低碳足迹：

1. **烹饪方式优化**：

   - 触发条件：烹饪能耗占比 >30%
   - 建议：改用低能耗方式（如蒸、煮），可减少 15-25%烹饪碳排
   - 示例：将"炸"改为"蒸"，能耗系数从 1.35 降至 1.05，减少 22%碳排

2. **季节选择优化**：

   - 触发条件：检测到反季食材
   - 建议：选择应季食材，可减少 20%季节调整碳排
   - 示例：冬季选择白菜（应季）替代番茄（反季），减少 0.2 的系数加成

3. **运输优化**：

   - 触发条件：运输碳排占比 >15%
   - 建议：优先选择本地食材（<100km），可减少 15%运输碳排
   - 示例：本地食材运输系数 1.0，跨省运输 1.15，进口食材 1.35

4. **保存方式优化**：

   - 触发条件：使用冷冻或罐装食材
   - 建议：优先选择新鲜食材，减少 5-15%保存碳排
   - 示例：新鲜食材保存系数 1.0，冷冻 1.10，罐装 1.15

5. **食材组合优化**：
   - 触发条件：菜品总碳足迹 >2.0 kg
   - 建议：增加低碳食材（叶菜、根茎类）占比，减少高碳食材用量
   - 示例：叶菜类系数 0.4，豆制品 1.2，合理搭配降低总碳排

**优化效果评估**：
系统会计算每个优化建议的潜在减排量，帮助餐厅评估优化效果和成本效益。

##### 3.1.2.8 计算方法应用说明

**计算模式选择**：

- **基础计算方法**：适用于快速估算、菜单预览、批量计算等场景，计算速度快，资源消耗低
- **高级计算方法**：适用于精确核算、认证审核、ESG 报告等场景，计算精度高，数据完整

**数据来源**：

- 食材碳足迹系数：主要来自`ingredients`数据库集合，支持动态更新
- 基准值设定：基于行业标准和历史数据，可自定义调整
- 系数表：系统内置，支持管理员配置更新

**计算精度**：

- 基础方法：精度 ±10%，适用于日常运营
- 高级方法：精度 ±5%，适用于精确报告
- 数据精度：保留 2 位小数，单位 kg CO₂e

**性能优化**：

- 批量计算：支持一次计算多道菜品，提升效率
- 缓存机制：常用食材系数缓存，减少数据库查询
- 异步计算：大批量计算采用异步处理，不阻塞用户操作

**数据验证**：

- 输入验证：食材重量、系数等参数范围检查
- 结果验证：计算结果合理性检查，异常值告警
- 审计日志：记录所有计算操作，支持追溯和审计

##### 3.1.2.9 碳数据来源与更新机制

**数据来源类型**：

- 国际权威数据库：Ecoinvent、IPCC、FAO 等公开碳排系数
- 本地农业与环保部门：各省市农技站、生态环境局的年度碳排数据
- 餐厅供应链合作伙伴：供应商上传的产地、加工、运输碳排系数
- 实地测算与第三方机构：与认证机构合作的定制测算结果

**数据采集流程**：

1. 数据管理员定期从权威数据库同步最新数据，生成导入文件
2. 供应商通过 B 端后台上传食材碳足迹资料（含原始凭证）
3. 系统自动校验字段完整性、数值范围、单位是否正确
4. 符合条件的数据进入审核队列，等待人工复核

**审核角色分工**：

- 数据管理员：负责初审，确认数据格式、来源合法性
- 碳审计专员：复核碳排系数的科学性，与行业标准对比
- 行业专家顾问（可选）：针对争议数据给出专业意见
- 系统管理员：审批通过后发布到生产环境，并更新版本号

**数据校验规则**：

- 单位标准化：统一换算至 kg CO₂e/kg、kg CO₂e/L 等
- 数值范围：与历史数据对比，超出 30% 波动需人工复核
- 元数据完整性：来源、采样时间、适用区域、置信度必须填写
- 逻辑关联：食材类别、加工方式需与字典表匹配

**更新节奏与版本控制**：

- 月度小更新：同步新食材、修正错误
- 季度大更新：整合国际数据库最新版本
- 特殊应急更新：政策调整或重大异常时即时发布
- 版本管理：采用语义化版本号（如 v2025.03），保留历史版本以供追溯

**异常处理与回滚**：

- 异常捕捉：监控计算结果波动，如单日碳排异常增加 >40%
- 问题定位：自动记录造成异常的数据版本、录入人、时间
- 快速回滚：一键恢复至上一稳定版本，并通知相关餐厅
- 补偿机制：对已发布的数据重新计算并同步修正后的碳足迹

**沟通与培训**：

- 向餐厅定期推送数据更新日志和使用指南
- 供应商培训：明确数据提交标准、模板与审核时间
- 内部培训：保持审核团队对最新行业标准的理解

##### 3.1.2.10 对外数据协同与接口方案

**合作对象与业务场景**：

- 第三方碳核算机构：共享核算模型、校准碳排系数、联合出具认证报告
- 政府碳普惠/生态环境平台：提交餐厅低碳数据，参与补贴、积分或政策评价
- 企业 ESG/供应链平台：提供低碳运营数据，支撑上下游的可持续披露需求
- 生态伙伴（配送、能源、回收企业）：对接运营数据，实现跨场景低碳协同

**接口类型与协议**：

- RESTful API：基于 HTTPS + JSON，用于实时推送、查询与回写
- 批量文件接口：支持 CSV/Excel/JSONL，通过加密 SFTP/OSS 进行定时批量同步
- 消息队列/事件订阅：Kafka、RabbitMQ 等方式处理高频事件与异步同步需求
- 标准化数据模型：遵循 ISO 14064、GB/T 32151、PAS 2050 等碳核算标准

**核心数据字段**：

- 餐厅信息：餐厅编码、认证等级、所在地、受理周期
- 菜品数据：菜品编码、碳足迹值、碳标签等级、核算版本
- 订单数据：订单号、消费时间、减排量、积分、匿名用户 ID
- 元数据：数据来源、核算模型、置信度、审核状态、更新时间

**安全与认证机制**：

- OAuth 2.0/JWT：外部系统凭授权令牌访问，支持动态权限吊销
- 双向 TLS：与政府平台建立证书互信，保障数据传输安全
- API 网关 + IP 白名单：限制访问来源，记录调用明细并支持限流
- 数据脱敏/加密：对个人信息、敏感字段进行哈希或字段级加密处理

**对接流程**：

1. 签署数据共享协议，明确数据范围、频次和责任边界
2. 交换测试环境凭证，在沙盒环境完成接口联调
3. 校对数据字段与格式，输出对接说明书与测试用例
4. 试运行阶段监测数据准确性、延时及异常情况
5. 正式上线并纳入日常监控与 SLA 管理

**数据回传与协同机制**：

- 第三方复核回写：接收外部核算机构返回的校准结果与报告编号
- 政策激励反馈：政府平台回传补贴、积分发放结果并自动入账
- 异常更正通道：外部平台触发更正接口，附带原因与证明材料
- 回执流水：所有交互生成唯一流水号，便于双方追溯和审计

**运维与监控保障**：

- 关键指标：调用成功率、响应时间、错误类型、流量峰值
- 自动化策略：配置熔断、限流、重试、降级，保障高峰稳定性
- 日志审计：保留 3-5 年访问日志与数据备份，满足监管要求
- 灾备演练：定期开展容灾切换、数据回放与应急演练

**合规治理要求**：

- 数据分级管理：按敏感度控制共享范围，遵循最小权限原则
- 法规遵循：满足《数据安全法》《个人信息保护法》及地方规范
- 第三方评估：定期邀请权威机构开展数据安全与合规评估
- 风险预警：建立对外接口风险台账，定期评估并发布预警

**界面设计**：

- **菜单管理页面**：菜品列表，显示碳足迹和碳标签
- **订单统计页面**：订单列表和统计图表
- **碳报告页面**：报告预览和下载

#### 4.2.3 多租户隔离与数据导出

- **命名空间策略**：平台统一托管“气候餐厅”数据库，为每家餐厅分配租户 ID，所有集合通过 `tenantId` 字段或分片键隔离，保障数据写入与查询互不干扰。
- **权限控制**：结合数据库 ACL 与应用层 Token，限制餐厅管理员仅能访问本租户数据；平台审核、监管账号依据最小权限原则获取跨租户视图。
- **公共数据复用**：碳排系数、标签规则、成长激励等公共数据存放在共享命名空间，全体租户共用，保持标准一致与版本可控。
- **数据导出接口**：提供按租户的标准化导出 API/批量下载能力，支持菜谱、食材、碳足迹、认证报告等数据导出，方便餐厅对接自有系统或本地备份。
- **备份与追溯**：平台定期生成租户级数据快照，支持按租户回滚、合规审计与政府报送的追溯需求。

#### 4.2.2 数据模型

**食材溯源链模型**：

```javascript
{
  _id: ObjectId,
  traceabilityId: String,        // 溯源链ID
  restaurantId: ObjectId,        // 餐厅ID
  menuItemId: ObjectId,         // 菜品ID
  ingredientId: ObjectId,        // 食材ID
  chain: [{
    nodeType: String,            // 节点类型：supplier/processor/restaurant
    nodeId: ObjectId,            // 节点ID
    nodeName: String,            // 节点名称
    timestamp: Date,             // 时间戳
    location: {                  // 位置信息
      latitude: Number,
      longitude: Number,
      address: String
    },
    certifications: [String],    // 认证信息
    carbonFootprint: Number      // 该节点碳足迹
  }],
  totalCarbonFootprint: Number,  // 总碳足迹
  trustScore: Number,            // 信任度评分 (0-100)
  createdAt: Date,
  updatedAt: Date
}
```

**用户消费行为模型**：

```javascript
{
  _id: ObjectId,
  userId: ObjectId,
  restaurantId: ObjectId,
  orderId: ObjectId,
  behaviors: [{
    type: String,                // 行为类型：low_carbon_choice/small_portion/no_utensils
    timestamp: Date,
    impact: {                    // 影响
      carbonReduction: Number,
      pointsEarned: Number
    }
  }],
  totalCarbonReduction: Number,
  totalPointsEarned: Number,
  createdAt: Date
}
```
