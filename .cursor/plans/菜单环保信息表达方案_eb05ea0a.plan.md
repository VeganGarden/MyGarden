# 菜单环

保信息表达方案

## 一、项目概述

### 1.1 目标

设计一套完整的菜单环保信息表达方案，帮助气候餐厅平台在不同媒介上清晰、直观地展示菜品碳足迹信息，引导消费者做出更环保的饮食选择。

### 1.2 适用范围

- 纸质菜单（打印菜单）
- 电子菜单（点餐屏幕/平板）
- 手机App/小程序
- 在线菜单（网站/外卖平台）
- 收银系统界面
- 小票打印

## 二、全球现状研究

### 2.1 国际实践案例

#### 研究案例

- **美国研究**：菜单生态标签使餐品碳足迹降低9.2%，同时增加膳食纤维摄入
- **交通灯系统**：红、黄、绿三色图标分别代表高、中、低碳排放（研究显示降低高碳排放餐品供应量）
- **价格激励**：瑞士研究显示对素食餐点提供折扣或对肉类加价，显著提高素食选择率

#### 实际应用案例

- **麦当劳中国**：北京首钢园"零碳餐厅"，展示发电量、用电量及相当于植树和减排的数值
- **外卖平台**：饿了么、美团等提供"无需餐具"环保选项
- **个性化推荐**：利用推荐系统优先推荐可持续食品选项

### 2.2 研究发现

- 生态标签可以有效引导消费者选择低碳餐品
- 视觉化标识（颜色编码）比纯文字更有效
- 结合价格激励可以进一步促进环保选择
- 需要简洁易懂，避免信息过载

## 三、现有系统分析

### 3.1 数据结构（已存在）

#### 碳标签数据（carbonLabel）

```javascript
{
  level: "low" | "medium" | "high",
  icon: "图标URL",
  description: "描述文字",
  qrCode: "二维码URL（可选）"
}
```



#### 碳足迹数据（carbonFootprint）

```javascript
{
  value: 0.45,              // 碳足迹值（kg CO₂e）
  unit: "kg CO₂e",          // 单位
  calculationLevel: "L2",   // 计算级别
  baseline: 1.2,            // 基准值
  reduction: 0.75,          // 减排量
  reductionPercent: 62.5,   // 减排百分比
  carbonLevel: "low"        // 碳等级
}
```



### 3.2 等级划分标准

- **low（低碳）**：碳足迹 < 基准值 × 0.8（显著减排）
- **medium（中碳）**：基准值 × 0.8 ≤ 碳足迹 ≤ 基准值 × 1.2（行业达标）
- **high（高碳）**：碳足迹 > 基准值 × 1.2（建议优化）

## 四、设计方案

### 4.1 视觉标识系统

#### 4.1.1 碳标签图标设计

**设计原则**：

- 简洁明了，易于识别
- 符合国际通用色彩编码（绿色=好，黄色=中，红色=警示）
- 适配不同尺寸和媒介

**图标方案**：

1. **低碳标识（low）**

- 图标：绿色叶子图标 🌱 / 绿色圆形带"低"字
- 颜色：绿色（#4CAF50 / #52C41A）
- 文字：可选显示"低碳"、"Climate Friendly"等

2. **中碳标识（medium）**

- 图标：黄色圆形图标 ⚪ / 黄色圆形带"中"字
- 颜色：黄色（#FFC107 / #FAAD14）
- 文字：可选显示"达标"、"Standard"等

3. **高碳标识（high）**

- 图标：橙色/红色圆形图标 🔴 / 橙色圆形带"高"字
- 颜色：橙色（#FF9800）或红色（#F5222D）
- 文字：可选显示"高碳"、"High Carbon"等
- 注意：避免过于负面，建议用"建议优化"而非"不环保"

#### 4.1.2 视觉元素规范

**最小尺寸要求**：

- 纸质菜单：12×12mm
- 电子屏幕：24×24px（1x）或 48×48px（2x）
- 移动端：20×20pt（iOS）或 20×20dp（Android）
- 在线菜单：24×24px

**间距规范**：

- 图标与菜品名称间距：8-12px
- 图标与价格间距：16-20px
- 多个标识之间的间距：8px

### 4.2 展示级别配置

#### 级别1：极简模式（Minimal）

**适用场景**：纸质菜单、空间受限的界面**展示内容**：

- 仅显示碳标签图标（不显示文字和数值）
- 图标位置：菜品名称右侧

**示例**：

```javascript
素麻婆豆腐        🌱      ¥38
```



#### 级别2：基础模式（Basic）

**适用场景**：标准纸质菜单、简单电子菜单**展示内容**：

- 图标 + 等级文字（中文：低碳/达标/高碳 或 英文：Low/Medium/High）
- 可选：减排百分比徽章

**示例**：

```javascript
素麻婆豆腐   🌱 低碳 (-62%)      ¥38
```



#### 级别3：详细模式（Detailed）

**适用场景**：电子菜单、移动端App、在线菜单**展示内容**：

- 图标 + 等级文字 + 具体数值（kg CO₂e）
- 可选：相比基准的减排量

**示例**：

```javascript
素麻婆豆腐   🌱 低碳  0.45 kg CO₂e   ¥38
             相比基准减排 62%
```



#### 级别4：完整模式（Comprehensive）

**适用场景**：详细页面、环保教育场景**展示内容**：

- 图标 + 等级 + 数值 + 基准对比 + 详细说明
- 可能包含：食材来源、烹饪方式影响等信息

**示例**：

```javascript
素麻婆豆腐   🌱 低碳排放菜品   ¥38
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
碳足迹：0.45 kg CO₂e
行业基准：1.20 kg CO₂e
减排量：0.75 kg CO₂e（减排62%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[查看详情] [食材来源]
```



### 4.3 多媒介适配方案

#### 4.3.1 纸质菜单

- 使用级别1（极简）或级别2（基础）
- 图标使用单色印刷（CMYK）
- 考虑印刷成本，建议批量印刷时统一添加
- 可在菜单底部添加图例说明

**图例示例**：

```javascript
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
碳标签说明：
🌱 低碳   ⚪ 达标   🔴 高碳
选择低碳菜品，为地球减碳贡献力量
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```



#### 4.3.2 电子菜单（点餐屏幕/平板）

- 使用级别2（基础）或级别3（详细）
- 支持点击查看详情
- 可使用动画效果（如低碳标签轻微闪烁或呼吸效果）
- 支持筛选功能（如"仅显示低碳菜品"）

#### 4.3.3 手机App/小程序

- 使用级别3（详细）
- 支持点击查看完整信息（级别4）
- 支持筛选和排序（按碳足迹排序）
- 可显示"今日推荐低碳菜品"专区

#### 4.3.4 在线菜单（网站/外卖平台）

- 使用级别3（详细）
- 支持交互式展示
- 可集成到第三方外卖平台
- 支持分享环保成就

#### 4.3.5 收银系统界面

- 使用级别2（基础）或级别3（详细）
- 订单汇总时显示总碳足迹
- 可选择是否在小票上打印

#### 4.3.6 小票打印

- 使用级别1（极简）或级别2（基础）
- 在订单底部显示：
  ```javascript
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        本订单碳足迹：0.90 kg CO₂e
        相比基准减排：1.50 kg CO₂e（62%）
        感谢您为环保做出的贡献！ 🌱
        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ```




### 4.4 特殊展示场景

#### 4.4.1 环保推荐专区

在菜单显著位置设置"今日低碳推荐"或"环保精选"专区，突出展示低碳菜品。

#### 4.4.2 对比展示

对于同一类菜品（如同为"麻婆豆腐"，但有肉版和素版），并排展示，直观对比碳足迹差异。

#### 4.4.3 成就激励

- 显示"您今日已减排 X kg CO₂e"
- 显示"相当于种植 X 棵树"
- 显示"本月环保之星"等激励信息

## 五、实施指南

### 5.1 配置管理

在餐厅管理后台提供菜单环保信息展示配置：

```javascript
{
  restaurantId: "rest_123456",
  menuDisplayConfig: {
    // 展示级别配置
    displayLevel: "basic", // minimal | basic | detailed | comprehensive
    
    // 各媒介配置（可覆盖全局配置）
    mediaConfig: {
      physicalMenu: "minimal",    // 纸质菜单
      digitalMenu: "basic",       // 电子菜单
      mobileApp: "detailed",      // 移动端
      onlineMenu: "detailed",     // 在线菜单
      receipt: "basic"            // 小票
    },
    
    // 显示内容配置
    showContent: {
      icon: true,              // 显示图标
      levelText: true,         // 显示等级文字
      value: false,            // 显示具体数值（kg CO₂e）
      reductionPercent: true,  // 显示减排百分比
      baseline: false,         // 显示基准值
      qrCode: false            // 显示二维码（链接到详情）
    },
    
    // 视觉样式配置
    style: {
      iconSize: "medium",      // small | medium | large
      colorScheme: "standard", // standard | colorful | minimal
      position: "afterName"    // beforeName | afterName | belowName
    },
    
    // 特殊功能
    features: {
      enableFilter: true,      // 启用筛选功能（仅显示低碳）
      enableSort: true,        // 启用排序功能（按碳足迹排序）
      enableRecommendation: true, // 启用推荐专区
      enableAchievement: true     // 启用成就激励
    }
  }
}
```



### 5.2 数据接口

利用现有的API接口，菜单环保信息已经包含在以下接口中：

1. **菜单推送接口**（平台 → 收银系统）

- 每个菜单项包含 `carbonLabel` 和 `carbonFootprint` 字段

2. **菜单查询接口**（前端 → 后端）

- 查询餐厅菜单时自动包含环保信息

3. **订单接口**

- 订单详情包含每个菜品的碳足迹信息
- 订单汇总包含总碳足迹和减排量

### 5.3 前端组件设计

需要开发以下前端组件：

1. **CarbonLabel组件**（碳标签图标）

- 输入：level, size, showText
- 输出：渲染的图标组件

2. **CarbonInfo组件**（碳信息展示）

- 输入：carbonFootprint, displayLevel
- 输出：根据配置级别渲染的信息

3. **CarbonFilter组件**（筛选组件）

- 支持按碳等级筛选

4. **CarbonComparison组件**（对比组件）

- 支持多个菜品碳足迹对比

### 5.4 图标资源

需要设计并准备以下图标资源：

1. **碳标签图标**（3个等级）

- low-carbon.png（绿色）
- medium-carbon.png（黄色）
- high-carbon.png（橙色/红色）
- 提供多种尺寸：16×16, 24×24, 32×32, 48×48, 64×64
- 提供SVG格式（矢量图，可缩放）

2. **辅助图标**

- 减排百分比徽章
- 环保推荐角标
- 二维码图标
